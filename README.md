# SynOrg-Analysis - Organoid Segmentation and Feature Extraction Pipeline

Companion analysis pipeline for the [SynOrg](https://github.com/DigitalSreeni/SynOrg) synthetic organoid generator. Segments 3D organoid images, extracts 36–38 per-cell morphological, intensity, and topology features, and runs condition-specific analyses.

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19616110.svg)](https://doi.org/10.5281/zenodo.19616110)

Companion to the manuscript:

> Bhattiprolu S. (2025). *Parametric Physics-Based Synthesis of 3D Fluorescence Organoid Images with Exact Ground Truth for Deep Learning Pipeline Development.* bioRxiv. [DOI placeholder]

---

## What it does

Five numbered scripts implement a sequential pipeline:

| Script | Purpose |
|---|---|
| `01-segment_organoids.py` | 3D cell and nucleus segmentation using Cellpose |
| `02-extract_features.py` | Per-cell feature extraction from segmentation masks |
| `03-hmecyst_topology_analysis.py` | Topology analysis: HMECyst control vs cyst-forming |
| `04-pdac_osmotic_stress_analysis.py` | Morphometric analysis: PDAC isotonic vs hypertonic |
| `05-pdac_large_spatial_heterogeneity.py` | Radial gradient and cluster analysis: large PDAC organoid |

Scripts 03–05 reproduce the three validation analyses described in the manuscript.

## Quick start

```bash
pip install -r requirements.txt

# Step 1 — segment organoids (reads from input/images/, writes to output/segmentation/)
python 01-segment_organoids.py

# Step 2 — extract features (reads masks, writes to output/features/)
python 02-extract_features.py

# Step 3 — run a condition-specific analysis
python 03-hmecyst_topology_analysis.py
python 04-pdac_osmotic_stress_analysis.py
python 05-pdac_large_spatial_heterogeneity.py
```

Steps 1 and 2 are independent — you can re-run feature extraction with different parameters (e.g. a different `RADIUS_UM`) without repeating segmentation.

## Extracted features

`02-extract_features.py` produces one CSV per organoid with 36 features (38 when border distance is enabled). Features are grouped into six categories:

| Category | Features |
|---|---|
| Metadata | Batch, Sample, Well, Field, Cell_Number |
| Spatial coordinates | Centroid (absolute, relative, normalised), radial distance, zone |
| Morphology | Cell volume, nucleus volume, elongation, roundedness, border distance (optional) |
| Nuclear intensity | Mean, std, max, min, sum DAPI intensity, CV_chromatin |
| Spatial topology | Neighbour count, neighbour distances, local density, crystal distance |
| 3D shape | Major, medium, minor principal axes, prolate ratio, oblate ratio |

See `Organoid_Feature_Reference_v2.pdf` for complete definitions, units, and analysis guidance for each feature.

## Key configuration parameters

Edit the `CONFIGURATION` block at the top of each script. The most important settings in `02-extract_features.py`:

| Parameter | Default | Description |
|---|---|---|
| `PROCESS_STEMS` | `None` | List of filename prefixes to process. `None` = all organoids |
| `RADIUS_UM` | `20.0` | Neighbourhood radius for topology features in µm |
| `COMPUTE_DISTANCE_TO_BORDER` | `True` | Compute nuclear border distance features |
| `READ_VOXEL_FROM_OME` | `True` | Read pixel size from OME-XML metadata |

## Input data

This pipeline is designed to work with:

- Synthetic organoids generated by [SynOrg](https://github.com/DigitalSreeni/SynOrg)
- Real organoid OME-TIFF images with two channels: channel 0 = DAPI/nuclear, channel 1 = Actin/membrane

Place raw OME-TIFF files in `input/images/` before running `01-segment_organoids.py`.
